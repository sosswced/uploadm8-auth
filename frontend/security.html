Security is a foundational commitment at UploadM8, not an afterthought. We apply defense-in-depth principles across every layer of the platform — from credential storage to API token handling to data transmission.

Security Controls

Encryption in Transit

All data transmitted between clients and UploadM8 servers uses TLS 1.3. Platform API communications use enforced HTTPS with certificate validation. No unencrypted connections accepted.

Encryption at Rest

All OAuth tokens, refresh tokens, and sensitive credentials are encrypted at rest using AES-256. Video files in Cloudflare R2 are stored with server-side encryption enabled by default.

Credential Isolation

Platform OAuth tokens are stored in isolated database rows accessible only to the publishing microservice. No other service component has direct token access. Tokens are never logged or exposed in error messages.

Audit Logging

Operational logs capture authentication events, publishing actions, administrative changes, and system events. Admin/system event tables use a rolling retention window (typically about six months) with automated purge of older rows.

Automated Backups

PostgreSQL databases are backed up on a 30-day rolling rotation. Backups are encrypted and stored in geographically separated storage. Restoration procedures are tested quarterly.

Role-Based Access Control

Internal team access follows least-privilege principles. Admin actions require multi-factor authentication. All privileged access events are logged with user attribution and timestamp.

Authentication Security

Password Storage

User passwords are hashed using bcrypt with an appropriate work factor. Plaintext passwords are never stored, logged, or transmitted. We do not have access to your password.

Session Management

Authentication sessions use cryptographically secure, randomly generated tokens. Sessions expire after inactivity. You can sign out on this device from the app header, or invalidate all other sessions via Settings → Security → Log Out All Other Sessions.

OAuth Token Lifecycle

Platform OAuth tokens are encrypted immediately upon receipt and decrypted only at the moment of API call execution. Tokens are automatically refreshed before expiration. Disconnecting a platform triggers immediate token revocation via the platform's OAuth revocation endpoint, followed by deletion from our database within 24 hours.

Incident Response

Detection & Response

We maintain automated monitoring for anomalous access patterns, failed authentication spikes, unusual API call volumes, and data exfiltration indicators. Security alerts route to on-call personnel 24/7.

Breach Notification

In the event of a security incident affecting your personal data, we will notify affected users via email within 72 hours of confirmation. Notifications will include: nature of the incident, data categories affected, likely consequences, and remediation steps taken.

Regulatory Reporting

Where required by applicable law (e.g., GDPR Article 33), we will report confirmed breaches to the relevant supervisory authority within the legally mandated timeframe.

Vulnerability Disclosure

We welcome security researchers who act in good faith to report vulnerabilities. Please follow responsible disclosure:

  • Email your findings to security@uploadm8.com with a detailed description
  • Allow us reasonable time to investigate and remediate before public disclosure (90 days standard)
  • Do not access, modify, or delete user data during your research
  • Do not conduct DDoS testing, social engineering, or physical security attacks
We will acknowledge receipt within 48 hours and provide a remediation timeline within 10 business days. Good-faith researchers will not face legal action for qualifying disclosures.

Your Security Responsibilities

We provide the infrastructure security. Your responsibilities include:

  • Using a strong, unique password for your UploadM8 account
  • Keeping your connected platform credentials (TikTok, YouTube, etc.) secure
  • Logging out from shared or public devices
  • Reporting suspicious account activity to security@uploadm8.com immediately
  • Not sharing your account credentials with third parties
  • Reviewing connected platform permissions periodically
